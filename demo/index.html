Eyas

Simplified hands-on testing for web applications. Bundle your app, share a file, let anyone test it.

How it works

1

Build your app

Run your normal production build (npm run build), then invoke Eyas CLI with one command.

→
2

Get a shareable file

Eyas bundles everything into a single .eyas database file — or a self-contained ZIP that needs no install.

→
3

Anyone can test it

Share the file. Recipients open it in the Eyas runner and get a full browser experience — no build tools, no servers.

🌐 Multi-env

Environment Selector

Configure dev, staging, and production domains. Eyas prompts you to choose on startup — or remembers your last choice.

 📐 Responsive

Viewport Testing

Switch between Desktop, Tablet, and Mobile presets, or define custom breakpoints in your config. Live readout on this page.

 🗺️ SPA Support

SPA Routing

Eyas serves files from your build directory and falls back to index.html for unmatched routes, just like a real CDN.

 🔗 Dynamic

Custom Links

Add shortcut links to your menu with variable substitution — integers, strings, booleans, picklists, and environment values.

 📡 Isolation

Network Control

Toggle the network on or off with one menu click. Test offline behaviour or isolate your app from external dependencies.

 🖼️ Resources

External Resources

 🔲 UX Polish

Window & Title

Eyas reflects your app's page title in the window title bar, including version and URL context. Open pages in new windows too.

 ↗️ Navigation

Multi-page Navigation

Navigate between pages just as your users would. Eyas fully supports multi-page apps built on traditional HTML routing.

Three ways to share

Pick the format that fits your workflow

One CLI, three outputs. Each method reaches a different audience.

npx eyas web 🌐

Web Link

Outputs a small eyas.json to your build directory. Deploy alongside your build artifact and share an eyas:// link. Best for teams with a staging server.

npx eyas db 💾

Database File

Bundles your entire build into a single .eyas file. Recipients open it with the installed Eyas runner. Roughly the same size as your production build.

npx eyas bundle 📦

Self-contained Bundle

Packages your app and the Eyas runner into one ZIP. No installation needed. Ideal for one-off reviews and external stakeholders. ~60–100 MB.

Simple config

Zero friction setup

Drop a single .eyas.config.js file at your project root. Eyas works without one too — sensible defaults out of the box.

// .eyas.config.js
module.exports = {
  source:  `dist`,
  title:   `My App`,

  domains: [
    { url: `dev.example.com`,  title: `Dev`     },
    { url: `stage.example.com`, title: `Staging` },
    { url: `example.com`,       title: `Prod`    },
  ],

  viewports: [
    { label: `iPad Pro`, width: 1024, height: 1366 }
  ],

  links: [
    { label: `Dashboard`, url: `{_env.url}/dashboard` },
    { label: `User #`,    url: `{_env.url}?userId={int}` },
  ],

  outputs: {
    expires: 168 // 1 week
  }
};
