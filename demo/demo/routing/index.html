How it works

Eyas's internal server acts like a real production web server. It's pre-configured to handle SPA catch-all routing out of the box — no complex Nginx or Docker configuration required for your tests.

  • ✔

    Path resolution: Checks your build directory for an exact match (e.g. /demo/path/).

  • ✔

    File resolution: Appends index.html to directory requests.

  • ✔

    SPA Fallback: If no file or directory is found, it serves the root index.html file. This allows client-side routers (like Vue Router or React Router) to take over and show a custom error page.
