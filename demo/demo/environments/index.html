Environment Selector

Eyas allows you to define multiple testing environments (like Dev, Staging, and Production) in your configuration. The runner will prompt you to select one on startup.

How it works

When you configure the domains array in your .eyas.config.js, Eyas creates a dedicated selection modal. This is perfect for QA teams who need to switch between different deployment stages quickly. 

// .eyas.config.js
module.exports = {
  domains: [
    { url: `dev.eyas.cycosoft.com`,     title: `Development` },
    { url: `staging.eyas.cycosoft.com`, title: `Staging`     },
    { url: `eyas.cycosoft.com`,         title: `Production`  }
  ]
};
Pro Tip: You can tell Eyas to "Remember this choice" so it skips the modal next time you open the same project. This can be toggled in the Settings menu.

Try it now

Go to the Test menu and select Reset Test Environment to see the selector in action.
